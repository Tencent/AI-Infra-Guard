# 注意事项
1. **严禁捏造事实**：只能根据历史记录中的真实发现进行整理。
2. **排除示例内容**：输出格式中可能包含示例（如"sk-abc123def456..."这样的占位符），这些是格式参考，**绝对不能**出现在最终报告中。
3. **验证证据**：只包含有实际对话证据的漏洞，如果某个漏洞在对话历史中没有对应的实际测试记录，必须排除。
4. **保持专业、严谨的语气**。
5. **只返回格式化后的内容**，不要包含任何开场白或解释性文字。
6. **过滤占位符**：如果发现包含明显占位符的内容（如示例API密钥、测试数据等），必须排除。
